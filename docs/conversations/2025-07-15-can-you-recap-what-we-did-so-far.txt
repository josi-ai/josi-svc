⏺ Summary: How Test Results Are Saved

  1. Multi-Level Storage

  The data is saved at multiple levels for different purposes:

  - Raw API Responses: Complete, unmodified API responses organized by endpoint
  - Individual Test Cases: All data for each test case in one file
  - Processed Data: Extracted values organized by type (planets, houses, etc.)
  - Validation Results: Comparison reports with accuracy metrics

  2. File Formats

  All data is saved as JSON files for:
  - Easy parsing and analysis
  - Human readability
  - Version control compatibility
  - Cross-platform support

  3. Data Organization

  test_data/vedicastro_api/
  ├── Raw responses (by endpoint/test case)
  ├── Collected data (by test case)
  ├── Processed data (by data type)
  └── Validation results (timestamped reports)

  4. Access Methods

  Direct JSON Access:
  import json
  with open('test_data/vedicastro_api/processed_data/planetary_positions.json') as f:

  from test_data_utilities import TestDataManager

  Export to CSV:
  python scripts/test_data_utilities.py
  # Creates CSV files for spreadsheet analysis

  5. Key Benefits

  - Traceability: Can trace any result back to original API call
  - Reusability: No need to re-fetch data for multiple validations
  - Analysis: Easy to analyze patterns and trends
  - Debugging: Raw responses help diagnose issues
  - History: Track accuracy improvements over time

  The system is designed to be comprehensive yet easy to use, allowing both automated validation and manual analysis of the test results.
