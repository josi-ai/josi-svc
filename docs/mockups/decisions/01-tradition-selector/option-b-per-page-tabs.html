Decision 1B: Per-Page Tradition Tabs

Each feature area has its own tradition context via inline tab bars. Users can view Vedic panchang while simultaneously seeing their Western chart. There is no global tradition state -- each card or section independently selects its tradition. This supports genuine cross-tradition exploration.

Some features are tradition-specific and only appear under the relevant tab: Panchang is Vedic-only, BaZi is Chinese-only, Hellenistic lots are Hellenistic-only. The tab bar naturally communicates this by only showing relevant traditions per feature.

Pros
  • Most flexible -- compare Vedic and Western side-by-side
  • No context switching needed for multi-tradition users
  • Tradition-specific features naturally surface where relevant
  • Encourages exploration and cross-tradition learning
  • Each section remembers its own last-selected tradition
Cons
  • More complex mental model -- "which tradition am I looking at?"
  • Repeated tab bars can feel visually noisy
  • More implementation complexity (per-section state)
  • Dashboard can feel fragmented with mixed tradition data
  • New users may not understand why tabs differ per section
