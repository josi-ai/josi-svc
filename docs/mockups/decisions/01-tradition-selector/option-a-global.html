Decision 1A: Global Tradition Switcher

A single dropdown in the sidebar header controls the tradition context for the entire application. When you switch from "Vedic" to "Western", every page, every card, every data point changes to reflect Western astrology. One tradition at a time.

This is the simplest mental model: "I am in Vedic mode right now." But it requires switching back and forth to compare traditions, and users who work across multiple traditions will find it cumbersome.

Pros
  • Simplest mental model -- one context, no confusion
  • Every feature auto-adapts (panchang only in Vedic, BaZi only in Chinese)
  • Sidebar nav items can change per tradition (show/hide tradition-specific pages)
  • Easiest to implement -- single global state
Cons
  • Cannot compare traditions side-by-side
  • Switching is disruptive -- losing context on current page
  • Users exploring multiple traditions must constantly toggle
  • Feels limiting for power users who think multi-traditionally
